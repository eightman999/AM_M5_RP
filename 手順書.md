# ロボット操作手順書

対象: `M5_AM_SEN0628_3xSEN0628_v15`(M5側 v23時点) + `AM_HighTorqueAntiStall_..._v4_for_x3`(Mega側 v2時点)

プロジェクト概要・配線・ソフトウェア構成は [README.md](README.md)、各定数の詳細は
[PARAMETERS.md](PARAMETERS.md) を参照。本書は実際に電源を入れてから走らせるまでの
操作手順に絞る。

---

## 1. 事前準備

- [ ] コース(通路)に前方の障害物が無いことを確認。特にキャリブレーション走行は
      直進で数メートル進むため、直線区間を確保する。
- [ ] Mega側・M5側それぞれの電源(バッテリー/USB)を確認。
- [ ] PCで点群ビューアーを使う場合は、M5のWi-Fi SoftAP(`M5_POINT_CLOUD`)へ
      後で接続できるようPCのWi-Fi設定を確認しておく。

## 2. 初回セットアップ(ファームウェア書き込み)

初回のみ、または.inoを変更した場合に実施。詳細はREADME.mdの「セットアップ」章参照。

1. Arduino IDEに以下のライブラリを導入:
   - M5StickCPlus
   - DFRobot_MatrixLidar
   - Seeed_BMA400 (`https://github.com/Seeed-Studio/Grove_3Axis_Digital_Accelerometer_BMA400`)
2. `M5_AM_SEN0628_3xSEN0628_v15/M5_AM_SEN0628_3xSEN0628_v15.ino` をM5StickC Plusへ書き込み。
3. `AM_HighTorqueAntiStall_500HzPWM_NoJumper_v4_for_x3/AM_HighTorqueAntiStall_500HzPWM_NoJumper_v4_for_x3.ino` をArduino Megaへ書き込み。

## 3. 起動手順

電源を入れると、M5のLCDに以下の順で初期化状況が表示される。

| 表示 | 意味 | 異常時の対応 |
|---|---|---|
| `INIT BMA400` → `OK` / `init NG` / `not found` / `TCA NG` | TCA CH0のBMA400初期化 | `not found`/`NG`でも走行自体は継続する(BMA400無しで動作)。配線・CH0接続を確認 |
| `INIT SEN` `1/3 CH1`〜`3/3 CH3` → `OK` / `not found` / `begin NG` | SEN0628(前/左/右)の初期化 | `not found`/`NG`が出たセンサーは以降無効。配線・TCAチャンネルを確認 |
| `TCA NOT` `FOUND` | TCA9548A自体が見つからない(致命的) | ここで完全停止する。I2C配線(SDA=32/SCL=33側)を確認して再起動 |
| `A: START` / `B: CAL` / `A+B: NVS RESET` / `Trim ±0.XX` | 起動完了、待機状態 | この画面が出れば操作可能 |

BMA400・SEN0628はここで失敗しても致命的ではない(該当機能が無効になるだけ)。
TCA9548A自体が見つからない場合のみ完全停止する。

## 4. キャリブレーション手順(BtnB)

**初回書き込み後、または車体・タイヤを変更した後は必ず実施する。**

1. 待機画面(`B: CAL`)でBtnBを押す。
2. `CAL STRAIGHT` / `Clear path!` / `Keep still...` が表示されている間はロボットを静止させておく
   (IMUオフセットを取得中、約1秒弱)。
3. 自動で複数ラウンドの走行が始まる。1ラウンドは短いパス(約3.3秒)×3回で構成される:
   - 各パス中は `ROUND {r} PASS {p}/3` を表示しながら最大速度で直進する。
   - パスとパスの間は約0.3秒停止する(この間に次のパスが自動で始まる。ロボットの位置は動かさない)。
4. 3パス終了後、そのラウンドの結果が表示される:

   | 表示 | 意味 |
   |---|---|
   | `R{n}: CAL OK` | ドリフト中央値が`CAL_OK_DEG`(4°)未満。収束成功 |
   | `R{n}: RETRY` | まだ収束していない。次ラウンドへ |
   | `Drift ±X.X deg (n/3)` | 中央値ドリフトと有効パス数 |
   | `Trim ±0.XXX` | 更新後のmotorTrim |
   | `CAL CANCELLED` | キャリブ中にBtnAを押して中止した場合 |

   **v23: 前方ToFによるコース不足チェックは廃止したため、各パスは必ずフル尺(約3.3秒)まで
   走行する。** コースの直線区間は運用者側で確保しておくこと(自動での早期中断はしない)。

5. `CAL OK` が出れば約2.4秒後に待機画面へ戻り、キャリブレーション完了。
6. `RETRY` の場合、`Reposition robot` / `B:continue A:stop` が表示される。
   - **ロボットを手動で開始位置まで戻す**(後方センサが無いため自動では戻らない)。
   - 戻したら **BtnB** を押して次ラウンドへ継続。
   - 中止したい場合は **BtnA** を押す(`CAL CANCELLED`表示後、待機画面へ戻る)。
7. 最大12ラウンド(v23: 5→12、モーター個体差2倍まで収束できるよう拡大)まで自動継続。
   それでも収束しない場合は `NOT CONVERGED` と表示され終了する。
   - この場合、`TRIM_LIMIT`(v23: ±0.34)に張り付いていないか確認し、張り付いていればハード側
     (モーター・ギア・摩擦の左右非対称)を疑う(PARAMETERS.md 1.6節参照)。

**v19: PC側点群ビューアーを起動していれば、キャリブレーション中も右側の「CALIBRATION」パネルに
ラウンド/パス番号・直近ラウンドのドリフト・motorTrimの値がリアルタイムに表示される
(LCDと同じ情報をPC画面でも確認できる)。**

## 5. 走行手順

**v20: 超信地旋回(その場旋回)を廃止し、常に前進しながら差動ステアだけで避ける方式に変更した。
実機での回避挙動確認はまだ行っていない。低速で、目を離さずに初回テストすること。**

1. 待機画面(`A: START`)で **BtnA** を押す。
2. `IMU ZERO` / `Keep still` の間は静止させておく。
3. `START` 表示後、自動走行が始まる(通路中央維持・障害物回避・コーナー旋回を自動判断。
   その場で完全停止して回転する超信地旋回はしないが、前方が危険域(frontRed)の間は
   片輪を止める急旋回(タンクターン、v22)で対応する)。
4. 走行中のLCD表示:
   - 状態: `RUN` / `BACK`(`PIVT`はv20で廃止)
   - 理由(`reason`): `FWD` / `FWD_CARE` / `AVOID_L` / `AVOID_R` / `TURN_L` / `TURN_R` /
     `ESCAPE_L` / `ESCAPE_R` / `BACK` / `CORRIDOR` など
     (`AVOID_*`=側方中心の危険での連続ステア中、`TURN_*`=前方が実際に危険域でのタンクターン中、
     `ESCAPE_*`=両側とも危険な状態でのタンクターン/連続ステア中)
   - Trim値、クリアランスなど
5. 走行を止めたい場合は物理的に持ち上げるか電源を切る(ソフトウェア側の緊急停止ボタンは無い点に注意)。

## 6. 点群ビューアーの使い方(PC側)

1. ロボット電源投入後、PCのWi-Fiを `M5_POINT_CLOUD` (パスワード `m5pointcloud`) に接続。
2. PCで以下を実行:
   ```
   python3 tools/point_cloud_viewer.py
   ```
3. 画面右側に状態(STATE/REASON/IMU・TARGET/CLEARANCE/MOTOR/CALIBRATION/MAP/LINK)が表示され、
   左側に点群マップ・推定軌跡・ロボットの向きが描画される。
4. `LINK`欄の `BMA400 OK`/`NG/WAIT` でBMA400の接続状況を確認できる。
   `CALIBRATION`欄はキャリブ中(STATE=CALIBRATING)にラウンド/パス/ドリフト/トリムを表示し、
   非キャリブ中は直近のtrim値のみ表示する。
5. 「マップをCSV保存」「マップを消去」ボタンで記録した点群を保存・リセットできる。
6. 動作確認だけしたい場合(実機不要):
   ```
   python3 tools/point_cloud_viewer.py --self-test
   ```

## 7. 終了手順

- ロボット・PC双方の電源/プロセスを終了するだけでよい(NVSに保存されたmotorTrimは
  次回起動時に自動復元される)。

## 8. トラブルシューティング

| 症状 | 確認ポイント |
|---|---|
| `TCA NOT FOUND`で起動しない | TCA9548Aの配線(外部I2C: SDA=32, SCL=33)、電源 |
| SEN0628が`not found`/`begin NG` | 該当TCAチャンネル(CH1/2/3)の配線、SEN0628のI2Cアドレス(0x33) |
| BMA400が`not found`/`init NG` | TCA CH0の配線、BMA400のI2Cアドレス(0x15)、Seeed_BMA400ライブラリ導入済みか |
| キャリブレ中に壁へぶつかりそうになる | v23で前方ToFによる自動中断を廃止したため、直線区間の長さは運用者側で確保する必要がある。足りない場合は`CAL_PASS_DRIVE_MS`を短くするか、コースを見直す(PARAMETERS.md参照) |
| キャリブレが`NOT CONVERGED`のまま収束しない | `TRIM_LIMIT`張り付きを確認。張り付いていればハード側の左右非対称を疑う |
| 走行中に左右へ流れる | まずBtnBキャリブレを再実行。改善しなければ`AUTO_TRIM_RATE`/`CAL_TRIM_GAIN`(v16で引き上げ済み、未検証)の振動有無を確認 |
| 点群ビューアーが「受信停止」表示のまま | PCのWi-Fi接続先が`M5_POINT_CLOUD`になっているか、UDPポート4210がファイアウォールで塞がれていないか |
| 点群ビューアーの軌跡が実際の動きとズレる | README.mdの「既知の課題・未検証事項」参照(BMA400軸の未実機検証など) |
| 障害物・壁に接触する/避けきれない(v20〜) | 超信地旋回廃止により旋回半径が大きくなっているため。v26で`FWD_CMD`/`FWD_SLOW_CMD`を140/120へ減速し、前方回避判定も120〜180mmへ早めた。まだ接触する場合はLCD/点群ビューアーで検出距離と回避方向を確認する(PARAMETERS.md 1.4/1.5節参照) |
| 前・左・右がすべて赤 | コーナー脱出として設定方向へ2000msカーブバックしてから、四角検知時に固定した90°目標まで残り角度だけ旋回する。バック終了角からさらに90°は回さない |
| 前方極近でバックする | v40以降は3000ms直線後退する。後方センサーがないため、通常走行でも後方空間を確保する |
| motorTrimの適用 | v29以降はBtnBキャリブでNVSへ保存された値だけを、キャリブ時・通常走行とも全量適用する。走行中の自動学習はしない。片輪停止/強旋回時だけは左右の旋回力を揃えるためtrimを適用しない |
| センサー視野の端にある障害物への反応が弱い | v30以降は前方8列すべてを使用し、3基それぞれの左右端2列を2セルから反応する専用判定で補強する。LCD/点群ビューアーで端の障害物に対応する方向が赤へ変わるか確認する |
| 床を障害物として拾う | v31以降は障害物判定を中央4行(row 2〜5)に限定し、床を拾いやすい上下各2行を走行判断から除外する。点群には診断用として全8行が引き続き表示される |
| 四角での周回方向 | `CLOCKWISE=true`なら右へ90°（時計回り）、`false`なら左へ90°（反時計回り）。前方赤でIMU基準90°旋回へ入り、全部赤なら同方向へ2000msカーブバックしてから90°旋回する |
| 瞬間的な前方赤で90°旋回する | v35以降は全点をXY変換し、前方壁と周回外側の側壁が作るL字形状＋異なる2フレーム＋200ms継続で四角確定する。細い障害物・単独壁・短い検知は通常回避になり、その後LCD `RETURN`で元の直進角度へ戻る |
| 後壁に密着した障害物を四角と誤認する | v36以降は前方壁に50mm区画で400mm以上の連続幅を要求する。最大幅300mmの障害物と、その左右に分断して見える後壁はL字コーナーから除外する |
| 幅約1mのコース中央で通路認識されない | v37以降は車幅約30cmから中央時のタイヤ外端クリアランスを片側約35cmと算出し、通路壁認識上限を420mmへ拡大している |
| 半透明プラ板の検知が弱い | v42以降は前方警戒を280mm、側方警戒を320mm、個別セル警戒を280mmへ拡大し、少数有効点の警戒スコアを50へ上げている。ただしToF光を透過する素材は常時検知を保証できない |
| コース中央の障害物で回避方向が定まらない | v43以降はL字でなく左右差が小さい前方赤を`CENTER_L/R`として、時計回りなら右、反時計回りなら左へ固定回避する。通過後は`RETURN`で元方位へ戻る |
| 中央障害物の規定回避側が狭い | v44以降は規定回避側のクリアランスが車幅300mm未満で、反対側に300mm以上ある場合、自動的に反対側へ逃がす |
| 左右表示が通常走行中も常に赤い | v45以降は左右の代表距離を160/250mm、左右8x8点群を130/230mmへ縮小した。前方の半透明板対策用160/280mmは維持する |
| 3秒バック後も前方が赤いのに前進する | v46以降はバック時間終了後も`BACK_HOLD`で停止し、前方赤の解除が300ms連続した場合だけ走行へ復帰する。後方センサーがないためバックの無期限延長はしない |
| 前方表示が障害物通過後も赤のまま残る | v47以降はSEN0628の64セル読取バッファを毎フレーム取得前に全消去する。センサーが一部セルを更新しなかった場合も前回の近距離値を再利用しない |
| バッファ消去後も前方だけ常時赤い | v48以降は消去済み一時バッファへ取得し、成功時だけキャッシュを更新する。前方判定は床側row 4〜5を外したrow 1〜3のみ使用する。左右判定はrow 2〜5のまま |
| LCDが`SENSOR_WAIT`のまま走らない | 3基のうち少なくとも1基が未初期化・測距失敗・620ms超の更新停止。安全のため欠測中は走行しない。LCD/Serial/点群ビューアーでfront/left/rightのどれが更新されていないか確認する |
