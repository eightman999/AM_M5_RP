# パラメーター解説

対象:
- `M5_AM_SEN0628_3xSEN0628_v15/M5_AM_SEN0628_3xSEN0628_v15.ino`（M5StickC Plus, 判断・センサー統合側, v16時点)
- `AM_HighTorqueAntiStall_500HzPWM_NoJumper_v4_for_x3/AM_HighTorqueAntiStall_500HzPWM_NoJumper_v4_for_x3.ino`（Arduino Mega, モーター駆動側, v2時点)

値はソース内の定義そのまま。**「調整の効果」列は挙動の傾向を説明するものであり、数値の正しさは実走行で確認すること。**特に「テスト値+40mm」と注記のある距離閾値は、卓上ベンチテストの実測値に安全マージンを足した運用値であり、コース(壁間隔・障害物サイズ)が変わればテストからやり直す前提。

---

## 1. M5側 (M5StickC Plus)

### 1.1 通信・センサー基本設定

| 定数 | 値 | 意味 | 調整の効果 |
|---|---|---|---|
| `AM_UART_BAUD` | 115200 | M5⇔Mega間UARTのボーレート | Mega側`Serial1.begin(115200)`と必ず一致させる |
| `TCA_ADDR` | 0x70 | I2Cマルチプレクサ(TCA9548A系)のアドレス | 実機のTCAアドレスが異なれば要変更 |
| `SEN_COUNT` | 3 | SEN0628の搭載数 | 増減時は`SEN_CH`/`SEN_ANGLE_DEG`/`senDev`配列も要同時変更 |
| `SEN_CH` | {1,2,3} | 各センサーのTCAチャンネル番号(front/left/right) | 実機配線に合わせる(CH0はBM予約で未使用) |
| `SEN_ANGLE_DEG` | {0, +50, -50} | 各センサーの水平マウント角(車体前方基準) | 前方/側方への幾何投影(`senCosAbs`/`senSinAbs`)に直結。実機取付角とズレると全ての距離判定が狂う |
| `SEN0628_ADDR` | 0x33 | SEN0628本体のI2Cアドレス | センサー側設定変更時のみ変更 |
| `POLL_SEQ` | {0,1,2,0,1,2} | センサーのポーリング順(v16: 前方偏重4:2:2→均等1:1:1に変更) | 増やす=そのセンサーの鮮度が上がる。前方の比率を下げすぎると`SENSOR_STALE_MS`との兼ね合いで前方安全マージンが減る可能性 |
| `AM_COMMAND_PERIOD_MS` | 16 | Megaへモーター指令を送る最小間隔 | 短くすると指令送信頻度は上がるが`sendDriveCommandSmooth`のslewは別途`CMD_SLEW_PER_SEND`で制御される |
| `AM_TIMEOUT_MS` | 300 | Megaからのテレメトリが途絶えたと判断するまでの時間 | Mega側`TELEMETRY_PERIOD_MS`(35ms)より十分大きく保つこと |
| `TEST_MODE` | false | 加速度診断(衝撃検知等)の計算・表示を有効化するフラグ | trueにすると`lastImpactG`等がTEST_MODE時のみ計算される(実走行の判断には使われない、表示専用) |

|---|---|---|---|

| `BODY_SIDE_MARGIN_MM` | 25 | 側方の当たり判定余裕(表示・内訳用) | 同上 |

### 1.3 走行速度

| 定数 | 値 | 意味 | 調整の効果 |
|---|---|---|---|
| `FWD_CMD` | 215 | 通常前進の最大デューティ | 上げると最高速が上がるがトルク余裕は減る |
| `FWD_SLOW_CMD` | 195 | 減速時の下限デューティ(v16: 段階的減速の下限として使用) | これより下げるとスキッドステア車体でストールしやすい(v8のコメント参照)。Mega側`MIN_EFFECTIVE_CMD`(132)より十分高い値を保つこと |
| `MIN_MOVING_CMD` | 150 | 前進の正方向指令の最低デューティ(底上げ) | Mega側`MIN_EFFECTIVE_CMD`(132)の直上に設定。旋回用の逆転/低速指令はこの底上げの対象外 |
| `MAX_MOVING_CMD` | 252 | 送信可能な最大デューティ(飽和防止で255未満) | キャリブ走行(`CAL_DRIVE_CMD`)にも使われる |
| `PIVOT_CMD` | 205 | 超信地旋回(正転逆転)時の左右デューティ | 上げると旋回が速くなるがIMU旋回角の検出精度・車体の安定性に注意 |

### 1.4 旋回(ピボット)判定

| 定数 | 値 | 意味 | 調整の効果 |
|---|---|---|---|
| `PIVOT_MIN_DEG` | 20° | 前方回避ピボットの最低回転角(これ以上回ってからでないと「前方が開いた」判定をしない) | 下げると小さい障害物への反応が早まるが行き過ぎ判定が甘くなる |
| `PIVOT_MAX_DEG` | 45° | 通常の障害物回避ピボットの絶対上限(Uターン化防止) | 上げると大きく避けられるが直進方位からの逸脱も増える |
| `PIVOT_MAX_DEG_CORNER` | 70°（v16新規） | コーナー継続旋回(直前ピボットが前方を開けられなかった場合の同方向継続)時の上限 | 90°コーナーを2回未満の継続ピボットで抜けられるように通常より拡大。上げすぎるとUターンに近くなるリスク |
| `ALIGN_PIVOT_DEG` | 18° | 片側極近時の姿勢直し小旋回の目標角 | 通路中央維持の微調整用。大きくすると姿勢直しが強くなる |
| `PIVOT_TIMEOUT_MS` | 1300ms | 前方回避ピボットの最大継続時間 | センサー不調等で終了条件が満たされない場合の保険。短すぎると回避不十分なまま強制終了する |
| `ALIGN_PIVOT_TIMEOUT_MS` | 650ms | 姿勢直しピボットの最大継続時間 | 同上(姿勢直し版) |
| `PIVOT_COOLDOWN_MS` | 300ms | 前方回避ピボットの再判定間隔 | 短くすると再ピボットまでの反応が速くなるが、連続ピボットで振動しやすくなる |
| `SIDE_ALIGN_DELAY_MS` | 400ms（v16: 1000→400） | 側方姿勢直しピボットの再判定間隔(前方回避直後の逆向き姿勢直しを抑止) | `PIVOT_COOLDOWN_MS`との非対称が大きいと側方姿勢直しが後回しになりがちだったため短縮。まだ大きい場合はさらに詰める余地あり |
| `COURSE_RETURN_DONE_DEG` | 3° | ピボット後、旋回前方位へ戻ったとみなす誤差 | 小さくすると復帰判定が厳しくなる(=`courseReturnActive`が長く残る) |
| `COURSE_RETURN_TIMEOUT_MS` | 2200ms | 旋回前方位への復帰を諦めるまでの時間 | 通路壁を再検知した場合はv16の変更でこのタイムアウトを待たず即座に復帰処理が打ち切られる(courseReturnActiveより corridorMode優先) |
| `CORNER_CONTINUE_WINDOW_MS` | 2000ms（v16新規） | 「直前の前方回避ピボットの延長」とみなす時間窓 | これを過ぎると通常のクリアランス比較/交互選択に戻る。コーナーが長く連続ピボットで抜けるのにこれより時間がかかる場合は延長が必要 |

### 1.5 前方/側方の距離閾値

| 定数 | 値 | 意味 | 調整の効果 |
|---|---|---|---|
| `FRONT_PIVOT_MM` | 90mm | 前方がこの距離未満(代表値)で回避ピボット開始 | テスト値50+40mm安全マージン。下げると障害物に近づいてから避けるようになる |
| `FRONT_CLEAR_EXIT_MM` | 110mm | ピボット中、前方がこの距離を超えたら「開いた」と判定 | `FRONT_PIVOT_MM`よりヒステリシス分大きい値にしてチャタリング防止 |
| `FRONT_SLOW_MM` | 120mm | 前方がこの距離未満で減速開始 | v16以降は`speedScore`の一因として連続的に効く(2値切替ではない) |
| `SIDE_PIVOT_MM` | 65mm | 側方クリアランスがこの距離未満で反対へ姿勢直し旋回 | テスト値25+40mm。下げると壁に寄ってから反応するようになる |
| `CLEARANCE_DIR_BIAS_MM`| 20mm（v16新規） | 前方回避ピボットの方向選択で「クリアランスが広い側」を優先する際の最小差 | 小さくするとわずかな差でも方向選択に反映されやすくなる(センサーノイズで方向がぶれるリスクも増える) |
| `FRONT_STOP_MM` | 60mm | 前方センサー実測の最小値がこの距離未満でBACK発動 | BACKの唯一のトリガー(v15でIMU衝撃トリガーは削除済み) |
| `FRONT_HARD_MM` | 75mm | 前方危険スコアが100%になる距離 | `obstacleScore`の危険側の境界値 |
| `FRONT_WARN_MM` | 100mm | 前方危険スコアが0%になる距離(これより遠いとスコア0) | `FRONT_HARD_MM`との差がスコアの傾き(反応の鋭さ)を決める |
| `SIDE_DANGER_MM` / `SIDE_WARN_MM` | 60mm / 80mm | 側方(投影)距離の危険/警戒境界 | 同上の側方版 |
| `SEN_DANGER_MM` / `SEN_WARN_MM` | 60mm / 80mm | SEN0628個別セルの危険/警戒境界(8x8全体のnearCount集計に使用) | 8x8のうち何セルが近いかでスコアを底上げする際の境界 |
| `SEN_DANGER_NEAR_COUNT` / `SEN_WARN_NEAR_COUNT` | 3 / 3 | 上記境界を超えるセル数がこの数以上でスコアを強制的に引き上げ | 下げるとノイズ1セルでも過敏に反応、上げると鈍感になる |
| `OBSTACLE_RED_SCORE` | 70 | LCD赤表示および実際の回避開始と共通のスコア閾値 | この値そのものが「危険」の定義。下げると早めに避け始める |
| `SEN_LOW_VALID_CAUTION_SCORE` | 40（v16: 18→40） | 有効測距点が少ない(信頼できない)場合に底上げする最低スコア | 減速閾値(35相当)を超える値にして、低反射率対象でも最低限減速するように変更 |
| `SEN_MOUNT_OFFSET_MM` | 0 | センサー取付面の凹み等の微調整用オフセット | v10で水平マウント化により上向きチルト補正は廃止。通常0のままでよい |

### 1.6 モーター左右差キャリブレーション

| 定数 | 値 | 意味 | 調整の効果 |
|---|---|---|---|
| `CAL_DRIVE_CMD` | `MAX_MOVING_CMD`(252) | BtnBキャリブ走行時の指令値(最大速度で差を計測) | 低速では非線形なズレが出るため意図的に最大速度固定 |
| `CAL_DRIVE_MS` | 10000ms | 1ラウンドの合計走行時間(旧来の単発10秒と同じ基準。v18でパス分割の正規化基準にも使用) | 長くすると測定精度は上がるが、コースの直線区間長に制約される |
| `CAL_ABORT_FRONT_MM` | 170mm | キャリブ走行中、前方がこの距離未満でアボート | TEST_MODEの閾値からv15で分離済み。コースの直線区間が短い場合はこの値を下げる必要がある(ただし安全マージンとのトレードオフ) |
| `CAL_PASSES_PER_ROUND`（v18新規） | 3 | 1ラウンドをこの回数の短いパスに分割し、中央値で外れ値を除去 | 増やすほどノイズに強くなるが1ラウンドの所要時間(パス間の待機込み)が伸びる |
| `CAL_PASS_DRIVE_MS`（v18新規） | `CAL_DRIVE_MS / CAL_PASSES_PER_ROUND`（約3333ms） | 1パスあたりの走行時間 | 1ラウンドの合計走行距離が旧来の単発10秒とほぼ同じになるよう自動計算 |
| `CAL_PASS_MIN_VALID_MS`（v18新規） | `CAL_PASS_DRIVE_MS / 2`（約1666ms） | これ未満で中断したパスは無効(コース不足扱い) | 短くすると多少短いパスでも採用されるが測定精度が落ちる |
| `CAL_MAX_ROUNDS`（v18新規） | 5 | `CAL_OK_DEG`に収束しない場合の自動打ち切り上限ラウンド数 | 増やすほど収束を粘れるが、収束しない個体差(ハード非対称)の発見が遅れる |
| `CAL_TRIM_GAIN` | 0.005（v16: 0.0035→0.005） | ヨードリフト1°あたりのトリム修正量 | 固定`LEFT_MOTOR_SCALE`廃止により初回キャリブで必要な補正幅が増えるため引き上げ。大きすぎると1回のキャリブで行き過ぎる(オーバーシュート)リスク |
| `TRIM_LIMIT` | 0.25 | `motorTrim`の絶対値上限(±25%) | この上限に張り付く場合はハード側(モーター/ギア/摩擦)の非対称を疑うべき |
| `CAL_OK_DEG` | 4° | この範囲のドリフト(中央値)なら「CAL OK」表示(合格) | 厳しくする(小さくする)と収束までのラウンド数が増えやすい |
| `AUTO_TRIM_ENABLE` | true | 走行中の`motorTrim`自動微学習を有効化 | falseにするとBtnBキャリブの結果のみに固定される |
| `AUTO_TRIM_RATE` | 0.00020（v16: 0.00004→0.00020） | 走行中の自動学習率(ヘディング補正量1あたり/周期) | 収束が遅すぎた旧値から5倍に引き上げ。振動的(ふらつき)になるようなら下げること |
| `TRIM_SAVE_PERIOD_MS` | 20000ms | 自動学習分をNVSへ定期保存する間隔 | 変化(差0.01超)があった時のみ実際に書き込む |

**v18のキャリブレフロー**: BtnB押下 → 1ラウンド(`CAL_PASSES_PER_ROUND`回の短いパス、パス間300ms待機)
→ 中央値ドリフトから`motorTrim`更新 → `CAL_OK_DEG`未満なら完了。未収束なら「Reposition robot / B:continue A:stop」表示で
ロボットを開始位置へ戻すよう要求し、BtnBで次ラウンドへ(後方無センサのため自動バックはしない)。
`CAL_MAX_ROUNDS`到達で「NOT CONVERGED」表示して終了。パス中にBtnA押下で即座にキャリブ全体を中止、
前方近接(コース不足)はそのラウンドを打ち切って有効パスのみで集計(0件なら「CAL FAILED」)。
**実機での収束確認・所要時間感はまだ検証していない。**

**v19: キャリブ情報のUDP配信**: `runStraightCalibration()`/`runOneCalPass()`実行中は
`dstate = DS_CALIBRATING`(値3)にし、`calStreamRound`/`calStreamPass`/`calStreamDriftDeg`を
更新しながら`streamPointCloud()`を呼ぶ(点群フレームプロトコルはv4に拡張、`trimMilli`/`calRound`/
`calPass`/`calDriftCdeg`フィールドを追加)。PC側`point_cloud_viewer.py`はCALIBRATIONパネルに
これをリアルタイム表示する。**実機でのM5→PC送信・表示は未検証(Python側デコードはセルフテスト済み)。**

### 1.7 バック(後退)判定

| 定数 | 値 | 意味 | 調整の効果 |
|---|---|---|---|
| `BACK_L_CMD` / `BACK_R_CMD` | -215 / -215 | バック時の左右デューティ | 前進と同じ大きさ・両輪逆転で直線バック |
| `FORCE_BACK_MS` | 400ms | 強制バックの継続時間 | 後方は無センサのため短めに制限。長くすると後方衝突リスクが上がる |
| `BACK_ARM_DELAY_MS` | 1200ms | 起動直後、BACKトリガーを有効化するまでの待機時間 | 起動時の強いトルク/通常加速をBACKと誤認しないための猶予(v15以降はBACKは距離センサーのみがトリガーなので、これは主に「起動直後の意図しない挙動」全般への保険) |
| `BACK_COOLDOWN_MS` | 2000ms | バック完了後、再バックまでの最短間隔（**v16でバグ修正: 以前は論理式の誤りで無効化されていた**） | 短くすると連続バックが起きやすくなる。長くすると本当に詰まった状況で再バックまで待たされる |
| `KICK_IGNORE_MS` | 450ms | 発進キック直後、衝撃検知を無効化する時間(TEST_MODE時のみ意味を持つ) | 実走行(TEST_MODE=false)には影響しない |

### 1.10 コマンドスムージング

コネクタは車体後ろ向きに実装(2026-07-22ユーザー確認)。Seeed製品ページの座標系シルク図
(Z=チップ面から手前=垂直上、Y=コネクタと反対方向、X=Y×Zの右手系)と照合すると、
水平・チップ面上向き取付の前提でX軸=車体右、Y軸=車体前方に一致するため、
`getAcceleration()`の生値(mg)はそのまま「右方向/前方向」として使っている
(符号反転・軸入替なし)。**この対応関係はWeb上の製品ページの図から導出したもので、
実機での重力方向テスト(静止時Z≒+1000mg等)による裏取りはまだ行っていない。**
PC側`point_cloud_viewer.py`ではM5内蔵IMUの加速度と単純平均でブレンドし、
どちらか片方しかフレッシュでなければそちらのみを使う(`bma_ok`フラグ判定)。

| 定数 | 値 | 意味 | 調整の効果 |
|---|---|---|---|

| 定数 | 値 | 意味 | 調整の効果 |
|---|---|---|---|
| `KICK_CMD` | 225 | キック中の出力デューティ | 上げると発進/転換の粘り抜けが強くなるが衝撃も強くなる |

| 定数 | 値 | 意味 | 調整の効果 |

## 3. パラメーター間の依存関係(要注意)

- **M5 `MIN_MOVING_CMD`(150) > Mega `MIN_EFFECTIVE_CMD`(132)**: M5側の底上げ済み指令が、Mega側の底上げを下回らないようにする関係。Mega側の値を変えたらM5側もそれに合わせて見直すこと。
- **M5 `FWD_SLOW_CMD`(195) > Mega `MIN_EFFECTIVE_CMD`(132)**: 減速時でも常にMega側の最低デューティを大きく上回っているため、通常の速度調整ではストールしない設計。
- **Mega `COMMAND_TIMEOUT_MS`(720ms) >> M5 `AM_COMMAND_PERIOD_MS`(16ms)**: 通常運用で誤タイムアウトしない十分なマージン。
- **M5 `AM_TIMEOUT_MS`(300ms) >> Mega `TELEMETRY_PERIOD_MS`(35ms)**: 同上、テレメトリ側。
- **`FRONT_PIVOT_MM`(90) > `FRONT_CLEAR_EXIT_MM`(110)は成立しない(90<110)**: これは意図的なヒステリシス。開始判定(90mm未満)と終了判定(110mm超)の間に20mmの不感帯があることでチャタリングを防いでいる。
- **`SEN_WARN_MM`(80) > `SEN_DANGER_MM`(60)**: `obstacleScore()`は`warnMm > dangerMm`前提の実装。逆転させると常時スコア0になるので注意。
- **`PIVOT_MAX_DEG_CORNER`(70) > `PIVOT_MAX_DEG`(45)**: コーナー継続旋回だけがこの拡張上限を使う。通常の障害物回避には適用されない(`pivotCornerMode`フラグで分岐)。

## 4. 実測に基づく外部情報(参考)

- **SEN0628実仕様**(DFRobot公表値): 8x8, 視野角60°, 測距3.5m, 20-200mm域で±11-12mm精度, 最大60Hz。0°/+50°/-50°の3基構成では隣接センサー間で約10°の重なりがあり、-80°〜+80°の範囲に角度的な死角はない。
- **車体**: スキッドステア2輪+前方キャスター。SEN0628マウントはボディ156(3MF内)で車体に対し水平取付。ただし車体自体は静止時に前方が1〜2°高くなる姿勢(全センサービームが実際の水平よりわずかに上を向く)。センサー高さ・壁高さの実測値がないため数値補正は未実装。
