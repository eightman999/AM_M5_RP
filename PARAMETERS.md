対象:
- `M5_AM_SEN0628_3xSEN0628_v15/M5_AM_SEN0628_3xSEN0628_v15.ino`（M5StickC Plus, 判断・センサー統合側, v16時点)

値はソース内の定義そのまま。**「調整の効果」列は挙動の傾向を説明するものであり、数値の正しさは実走行で確認すること。**特に「テスト値+40mm」と注記のある距離閾値は、卓上ベンチテストの実測値に安全マージンを足した運用値であり、コース(壁間隔・障害物サイズ)が変わればテストからやり直す前提。

### 1.4 連続ステア回避判定(v20: 超信地旋回を全廃止)

v16まではLCD赤判定/距離閾値で超信地旋回(その場で停止して回転)を開始し、IMU旋回角で終了判定する
方式だった。v20でこれを全廃止し、**常に前進しながら**危険度スコア(0-100)に応じて目標ヨー
(`targetYawDeg`)を連続的に「広い側/安全な側」へバイアスする方式に一本化した。実際の差動出力は
既存の`headingCorrection()`(±`HEADING_CORR_LIMIT`)がそのまま担うため、新規の差動計算は無い。

| 定数 | 値 | 意味 | 調整の効果 |
|---|---|---|---|
| `STEER_MAX_DEG` | 35°（v20新規） | 最大緊急度(avoidUrgency=100)でのヨーバイアス量 | `HEADING_CORR_LIMIT`(28)を飽和させる値に設定(35×`HEADING_KP`1.30=45.5>28)。下げると最大回避時の実効差動が弱まる |
| `STEER_URGENCY_MIN` | 15（v25: 18→15） | このスコア(0-100)未満は回避バイアスをかけない(通常走行/通路中央維持のみ) | v25で小さな左右差へ少し早く反応するよう変更 |

方向選択の優先順位(v16の考え方を継承): (1) 片側だけ危険(スコア≧`OBSTACLE_RED_SCORE`)なら必ず反対へ、
(2) `leftClear`/`rightClear`の差が`CLEARANCE_DIR_BIAS_MM`を超えれば広い側へ、(3) スコア差があれば
危険度が低い側へ、(4) 判断材料が無ければ直前にステアした方向(`lastSteerDir`)を維持。

**旧`DS_BACK`(前方極近の緊急バック)は維持。** v27以降は前・左・右のスコアがすべて
`OBSTACLE_RED_SCORE`以上になった場合も、空いている旋回方向がないため即バックする。
直前のバックから`BACK_COOLDOWN_MS`以内に三方向赤が続いている場合は、前進へ戻らず
`ALL_RED_WAIT`で停止し、クールダウン後も三方向赤なら再度バックする。
バック終了後はピボットせずそのまま前進へ戻り、
連続ステアが自然に避ける前提。バック時にどちらが広かったかは`backRecoverDir`として
`lastSteerDir`の初期値に使われる。

v33以降、前方赤を四角到達として、`CLOCKWISE`で指定した方向へIMU基準で90°旋回する。
三方向赤の場合は同じ設定方向へ2000msカーブバックしてから90°旋回へ接続する。
LCD reasonは時計回りなら`BACK_ARC_R`→`CORNER_R`、反時計回りなら
`BACK_ARC_L`→`CORNER_L`と遷移する。前方極近だけの通常BACKも3000msの直線後退を行う。
カーブバック前のヨーから最終90°目標を固定するため、バック中に回った角度を含めた
合計旋回量が90°になる。バック終了角から追加で90°回すことはない。

v35以降は中央4行の各点を`x=L*cos(theta), y=L*sin(theta)`で車体座標へ変換する。
v36以降は前方を横切る点群が横方向へ400mm以上連続する壁と、
周回方向外側に続く前後長50mm以上の側壁が
推定交点で接続するL字形状を四角候補とする。さらに異なる前方測距フレーム2回と200ms継続の
両条件を満たした場合だけ90°旋回する。それ以外は通常回避として扱う。
通常回避では開始前の直進ヨーを保存し、障害物消失後に`RETURN`状態でその角度へ戻る。

v43以降、L字形状ではなく、左右スコア差15以内・クリアランス差50mm以内の前方赤は
`CENTER_OBSTACLE`とする。時計回りなら右、反時計回りなら左へ35°まで変針しながら前進し、
通過後は`RETURN`で元の直進方位へ戻る。90°コーナーTURNには入らない。

**実機での回避挙動確認はまだ行っていない。** 超信地旋回よりその場での方向転換能力が大幅に
落ちる(差動ステアの実効旋回半径は`HEADING_CORR_LIMIT`とFWD速度で決まり、その場旋回よりずっと大きい)
ため、狭いコース・直角コーナーでは壁に接触するリスクがある。上記のFWD_CMD/FWD_SLOW_CMD減速は
その対策だが、それでも足りない場合はさらなる減速、`STEER_MAX_DEG`/`STEER_URGENCY_MIN`の調整、
または`HEADING_CORR_LIMIT`自体の引き上げを検討すること。

**v22: 前方タンクターン(frontRed時のみ)。** 上記の懸念(継続ステアだけでは前方の壁を
曲がりきれない)への対策として、`frontScore >= OBSTACLE_RED_SCORE`(=frontRed)の間だけ、
片輪を0、もう片輪を`TURN_CMD`にする急旋回(タンクターン)を優先実行する。逆転はしない
(超信地旋回はしない)ため「常に前進」の方針とは矛盾しない。前方が危険域を脱するとreason:
`TURN_L`/`TURN_R`(両側危険なら`ESCAPE_L`/`ESCAPE_R`)から自然に(2)の継続ステアまたは
通常前進へ戻る。専用の定数(タイムアウト・角度目標等)は無く、`frontRed`が続く限り
毎周期このタンクターンを再評価するだけの単純な仕組み。**実機での効き確認はまだ行っていない。**

### 1.5 前方/側方の距離閾値

| 定数 | 値 | 意味 | 調整の効果 |
|---|---|---|---|
| `FRONT_SLOW_MM` | 320mm（v42: 200→320） | 前方がこの距離未満で減速開始 | 増速と半透明板の弱反射を考慮し、早めに減速 |
| `CLEARANCE_DIR_BIAS_MM`| 20mm（v16新規） | 前方回避ピボットの方向選択で「クリアランスが広い側」を優先する際の最小差 | 小さくするとわずかな差でも方向選択に反映されやすくなる(センサーノイズで方向がぶれるリスクも増える) |
| `FRONT_STOP_MM` | 60mm | 前方センサー実測の最小値がこの距離未満でBACK発動 | BACKの唯一のトリガー(v15でIMU衝撃トリガーは削除済み) |
| `FRONT_HARD_MM` | 180mm（v42: 120→180） | 前方危険スコアが100%になる距離 | 半透明板へ余裕を持って強く反応 |
| `FRONT_WARN_MM` | 280mm（v42: 180→280） | 前方危険スコアが0%になる距離 | 180〜280mmで連続的に回避を強める |
| `SIDE_DANGER_MM` / `SIDE_WARN_MM` | 160mm / 250mm（v45） | 側方(投影)距離の危険/警戒境界 | 幅1mコースで左右が常時赤になるのを抑える |
| `SEN_DANGER_MM` / `SEN_WARN_MM` | 160mm / 280mm（v42） | SEN0628個別セルの危険/警戒境界 | 半透明板の少数戻りを遠めからスコアへ反映する |
| `SEN_DANGER_NEAR_COUNT` / `SEN_WARN_NEAR_COUNT` | 3 / 3 | 上記境界を超えるセル数がこの数以上でスコアを強制的に引き上げ | 下げるとノイズ1セルでも過敏に反応、上げると鈍感になる |
| `SEN_EDGE_NEAR_COUNT` | 2（v30新規） | 各8x8センサーの左右端2列で危険/警戒と判定する最小セル数 | 視野の継ぎ目と細い障害物は占有セルが少ないため、全体判定の3セルより早く反応させる |
| `SEN_ACTIVE_ROW_FIRST` / `SEN_ACTIVE_ROW_LAST` | 2 / 5（v31新規） | 障害物判定に使う8x8の縦方向行範囲 | 床・車体を拾いやすい上下各2行を走行判断から除外し、中央4行だけを使用する。raw取得と点群配信は全8行を維持 |
| `OBSTACLE_RED_SCORE` | 60（v28: 70→60、3/5） | LCD赤表示および実際の回避開始と共通のスコア閾値 | より遠い位置から赤判定・回避を開始する |

v30以降、前方センサーは中央6列だけでなく全8列を回避判定へ使用する。また3基すべてで
左右端2列を専用集計し、警戒距離内が2セル以上なら最低65点、危険距離内なら最低90点とする。
これによりセンサー視野の端・継ぎ目にだけ映った障害物も赤判定へ到達する。

さらに点群ビューアーと同じ水平角モデル
`SEN_ANGLE_DEG[sensor] + 26.25° - col×7.5°`で全192セルを展開し、前方
`-32〜+32°`、左`+18〜+80°`、右`-80〜-18°`へ重なり付きで再集計する。
前方と左右を14°重ねるため、3基の物理視野が約10°重なる部分も複数方向の回避判断へ反映される。
v31以降、この角度統合を含む障害物判定は中央4行(row 2〜5)だけを使用する。
上下端行のデータは診断用raw/点群には残るが、モーター制御には影響しない。
| `SEN_LOW_VALID_CAUTION_SCORE` | 50（v42: 40→50） | 有効測距点が少ない場合の最低スコア | 半透明・低反射対象で有効セルが減った場合も強めに減速する |
| `SEN_MOUNT_OFFSET_MM` | 0 | センサー取付面の凹み等の微調整用オフセット | v10で水平マウント化により上向きチルト補正は廃止。通常0のままでよい |

### 1.6 モーター左右差キャリブレーション

## 3. パラメーター間の依存関係(要注意)

- **M5 `MIN_MOVING_CMD`(110) > Mega `MIN_EFFECTIVE_CMD`(100)**: M5側の底上げ済み指令が、Mega側の底上げを下回らないようにする関係。Mega側の値を変えたらM5側もそれに合わせて見直すこと。
- **M5 `FWD_SLOW_CMD`(120) > Mega `MIN_EFFECTIVE_CMD`(100)**: 減速時でもMega側の最低デューティを上回る。実機でストールする場合は両者を同じ幅で少しずつ上げる。
- **Mega `COMMAND_TIMEOUT_MS`(720ms) >> M5 `AM_COMMAND_PERIOD_MS`(16ms)**: 通常運用で誤タイムアウトしない十分なマージン。
- **M5 `AM_TIMEOUT_MS`(300ms) >> Mega `TELEMETRY_PERIOD_MS`(35ms)**: 同上、テレメトリ側。
- **`SEN_WARN_MM`(280) > `SEN_DANGER_MM`(160)**: `obstacleScore()`は`warnMm > dangerMm`前提の実装。逆転させると常時スコア0になるので注意。
- **`STEER_MAX_DEG`(35)×`HEADING_KP`(1.30) > `HEADING_CORR_LIMIT`(28)**: 最大緊急度で確実に`HEADING_CORR_LIMIT`を飽和させる関係。`STEER_MAX_DEG`を下げすぎると最大回避時でも差動が頭打ちにならず、常に弱い回避力しか出せなくなる。

## 4. 実測に基づく外部情報(参考)

- **SEN0628実仕様**(DFRobot公表値): 8x8, 視野角60°, 測距3.5m, 20-200mm域で±11-12mm精度, 最大60Hz。0°/+50°/-50°の3基構成では隣接センサー間で約10°の重なりがあり、-80°〜+80°の範囲に角度的な死角はない。
- **車体**: スキッドステア2輪+前方キャスター。SEN0628マウントはボディ156(3MF内)で車体に対し水平取付。ただし車体自体は静止時に前方が1〜2°高くなる姿勢(全センサービームが実際の水平よりわずかに上を向く)。センサー高さ・壁高さの実測値がないため数値補正は未実装。
